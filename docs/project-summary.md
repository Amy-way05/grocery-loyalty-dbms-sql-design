# Project Summary

## Objective

Design and analyze a relational database for a grocery-store loyalty ecosystem. The project connects retail customers, loyalty tiers, rewards, promotions, stores, locations, and customer reviews into a query-ready DBMS structure.

## Quantitative Scope

- 8 relational tables.
- 1,307 total records across the operational model.
- 1,000 customer records.
- 212 loyalty accounts.
- 36 promotional campaigns.
- 28 rewards.
- 20 customer reviews.
- 5 stores across 3 locations.
- 3 loyalty tiers.

## Business Value

The database supports retail decision-making across:

- Loyalty program participation.
- Customer segmentation.
- Store-level promotion planning.
- Reward portfolio management.
- Review activity monitoring.
- Location-based performance reporting.

## Technical Value

The project demonstrates:

- ERD design.
- Relational schema planning.
- SQL joins.
- Aggregations and grouped reporting.
- Data dictionary documentation.
- Query organization for repeatable business analysis.
